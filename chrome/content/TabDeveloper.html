Tabulator: Developer notes

These are notes for people developing tabulator software. They are not designed for people developing websites using the tabulator library.

AJAX Specs, reference, tutorials and random notes

Semantic Web data (RDF)

RDF is the basic data format for semantic web data. N3 is an alternative format for the same data.

Notation 3 Tutorial An introduction to the semantic web using N3. The tabulator doesn't yet read N3 but we write in it and it is easier to understand than the RDF/XML form.

Notation3 home page

N3 Resources, including javascript N3 parser work EulerMoz which was:Samizdat

There is a set of six documents (Primer, Concepts, Syntax, Semantics, Vocabulary, and Test Cases) intended to jointly replace the original Resource Description Framework specifications, RDF Model and Syntax (1999 Recommendation) and RDF Schema (2000 Candidate Recommendation) developed by the RDF Core Working Group as part of the W3C Semantic Web Activity .

Debuging Javascript

Tips: always open the Javascript console on your browser.

Tip: Opera's exceptioon strings give a stack trace! This is much more valuable than Firefox's isolated message.

Tip: There is a way to get a whole extr debug menu in Safari. For mac do the following to enable Safari debug menu and hence debug window

defaults write com.apple.Safari IncludeDebugMenu 1

Venkman JavaScript Debugger project page. Venkman doesn't run under the latest Firefox but may be worth hanging back for it. It is a full debugging environment.

Venkman for Firefox 1.5 | Getahead - a site with a hacked Venkman for FF 1.5 before it came out

Javascript lint May be useful to check for undeclared variables assumed global, my least favorite JS bug. Source compiles (make -f Makefile.rdf, look for binary in machine-sepcific subdirectory) on OSX straight away.
